Why Only C/C++ Works

ReasonDetails
Dual architectureMust compile natively for SPARC and x86-64. Go/Rust/.NET have no or limited SPARC support.
Zero runtimeLinks only against libc + libsocket (always present). No JVM, .NET CLR, or Go runtime needed on endpoint.
All OS APIs are Ckstat, DTrace, ZFS, Zones, SMF, Doors, Event Ports — all C interfaces. Zero wrappers needed.
ABI guaranteeOracle guarantees C binaries from Solaris 10 run unchanged on 11.4. No other language has this.
Small footprint15–50 MB agents vs 100–500 MB for managed-language equivalents. Critical for Zones.
No GC pausesAgents must have predictable, low-latency performance. C has no garbage collector.
Cross-platformSame codebase with #ifdef __sun compiles for Solaris, AIX, HP-UX, Linux, macOS.

Architecture & Communication Model Relay hierarchy ▼

Agent runs as SMF service BESClient. Communication follows a relay hierarchy: Root Server → Relay(s) → Agent. Agents evaluate "Fixlet" relevance locally — only report back when a condition changes. Default heartbeat ~15 minutes. Protocol: HTTP/HTTPS (port 52311).

Cross-platform DSLs:

Relevance Language — Declarative query engine. name of operating system returns "SunOS" on Solaris. Under the hood, translates to uname(), kstat_open(), pkg list parsing via C function calls.

ActionScript — Imperative remediation engine. Package operations become pkg install (IPS) or pkgadd (SVR4). Script execution via sh -c. Same ActionScript template works across Windows, Linux, Solaris, AIX.

Patch Management — Boot Environment Rollback Unique in market ▼
BigFix is the only product that creates a ZFS Boot Environment before patching. If patching fails, instant rollback via beadm activate <old_BE>. No competitor implements this.

1. HCL maintains a "Patches for Solaris 11" Fixlet site mapping Oracle SRU releases to Fixlets.

2. Download plug-in authenticates with Oracle’s support repo. Air-gapped: Solaris Download Cacher tool.

3. IPS Repository Management dashboard configures local mirrors to reduce download time.

4. Agent creates Boot Environment → pkg update → verify → rollback on failure.

5. Relevance expressions verify installed SRU matches target. Non-compliant endpoints reported.

Compliance: DISA STIG Solaris 11 V3R2 (SPARC + X86) updated June 2025. CIS benchmarks available. All native C++ — no JRE required.

Installation on Solaris 11 .p5p since v9.5.13 ▼

Prereq: SUNWlibC package.

Standard install: pkg install -g BESAgent-*.p5p BESagent

Global zone with children: Create local repo → pkgrepo create /var/opt/BESClient_solaris_repo → pkgrecv -s BESAgent-*.p5p -d repo BESagent → pkg set-publisher --search-first -p repo

Upgrade: Self-updating via Fixlet. Config: /etc/opt/BESClient/actionsite.afxm

Architecture — Linear Chain P2P<15 sec for millions▼

Agents form a dynamic linear chain. Server sends query to leader agents → leaders forward to neighbors → neighbors forward to theirs. Results aggregate back up the chain with summarization at each node. On Solaris, uses POSIX sockets (socket(), connect(), send(), recv()) over TCP.

Modular design: Single core binary + signed Client Extensions (CX). Each module (Discover, Asset, Comply, etc.) ships as a .so shared object or script package. Agent verifies digital signatures before loading.

Section 07

Feature Gaps — What Nobody Built

Available on Windows/Linux in all three products but none implement on Solaris:

High Opportunity

EDR / Threat Detection

No competitor provides EDR on Solaris.

  • API: DTrace USDT probes — app/kernel tracing
  • API: Solaris Audit (BSM) — security event trail
  • Detect privilege escalation, unauthorized access
High Opportunity

Real-time File Integrity Monitoring

Tanium has hash-based; none do real-time kernel events.

  • API: port_associate(PORT_SOURCE_FILE)
  • Kernel-level — Solaris equivalent of inotify
  • Required for PCI DSS, HIPAA, SOX
High Opportunity

DTrace-based Monitoring

Solaris’s most powerful observability tool — untapped.

  • API: libdtrace C API
  • USDT probes — zero overhead when disabled
  • Production-safe application tracing
High Opportunity

Zone Lifecycle Management

All detect; none manage create/clone/migrate/delete.

  • APIs: zonecfg, zoneadm, libzonecfg
  • Primary Solaris virtualization
High Opportunity

ZFS Snapshot Management

No competitor manages ZFS from their console.

  • API: libzfs C API
  • Trigger, schedule, manage snapshots/clones
Medium Opportunity

USB Device Control

Available on Win/Linux — absent on Solaris.

  • APIs: devfsadm, syseventd
  • Different device model; dedicated work needed
