HCL BigFix

BigFix Inc. (1997) → IBM (2010) → HCL (2018). Most feature-complete Solaris agent in the market.

Architecture & Communication▼

Agent runs as SMF service BESClient. Relay hierarchy: Root Server → Relay(s) → Agent. Agents evaluate Fixlet relevance locally — report only on state change. Heartbeat ~15min. HTTP/HTTPS port 52311.

Relevance Language — Declarative queries. name of operating system → "SunOS". Translates to uname(), kstat_open(), pkg list via C.

ActionScript — Imperative remediation. Package ops → pkg install (IPS) / pkgadd (SVR4). Scripts via sh -c.

Patch Management — Boot Environment Rollbackunique in market▼

1. HCL maintains "Patches for Solaris 11" Fixlet site mapping Oracle SRUs to Fixlets

2. Download plug-in authenticates with Oracle repo. Air-gapped: Solaris Download Cacher tool

3. IPS Repo Management dashboard configures local mirrors

4. Agent creates ZFS Boot Environment before patching → pkg update → rollback via beadm activate on failure

5. Relevance verifies SRU level matches target

Compliance: DISA STIG Solaris 11 V3R2 (SPARC + X86) updated June 2025. CIS benchmarks available. All native C++ — no JRE.

Installation on Solaris 11.p5p since v9.5.13▼

Prereq: SUNWlibC. Install: pkg install -g BESAgent-*.p5p BESagent

Global zone w/ children: Create local repo first → pkgrepo create → pkgrecv → pkg set-publisher --search-first

Upgrade: Self-updating Fixlet. Config: /etc/opt/BESClient/actionsite.afxm

Architecture — Linear Chain P2P<15 sec for millions▼

Agents form dynamic linear chain. Server → leaders → neighbors → chain propagation. Results aggregate with summarization at each node. POSIX sockets on Solaris. Modular: Single binary + signed Client Extension (CX) .so files per module. Digital signature verification before loading.

Section 07

Feature Gaps

Available on Windows/Linux in all three — none built on Solaris:

High Opportunity

EDR / Threat Detection

No competitor provides EDR on Solaris.

  • API: DTrace USDT probes — app/kernel tracing
  • API: Solaris Audit (BSM) — security event trail
  • Detect privilege escalation, unauthorized access
High Opportunity

Real-time FIM

Tanium has hash-based; none do real-time kernel events.

  • API: port_associate(PORT_SOURCE_FILE)
  • Kernel-level — Solaris equivalent of inotify
  • Required for PCI DSS, HIPAA, SOX
High Opportunity

DTrace Monitoring

Solaris's most powerful tool — untapped by all competitors.

  • API: libdtrace C API
  • USDT probes — zero overhead when disabled
  • Production-safe application tracing
High Opportunity

Zone Lifecycle Management

All detect; none manage create/clone/migrate/delete.

  • APIs: zonecfg, zoneadm, libzonecfg
  • Primary Solaris virtualization technology
  • Full lifecycle from endpoint console
High Opportunity

ZFS Snapshot Management

No competitor manages ZFS from their console.

  • API: libzfs C API
  • Trigger, schedule, manage snapshots/clones
  • Integrate with patching for rollback
Medium Opportunity

USB Device Control

Available on Win/Linux — absent on Solaris.

  • APIs: devfsadm, syseventd
  • Different device model — dedicated work needed
